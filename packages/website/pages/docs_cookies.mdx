# Cookies

Cookies are perfect for storing auth, themes and other user preferences because they are available on both the server and client.

We made it easy to work with cookies by supporting bi-directional synchronization. If you modify a cookie on the server all the UI that is observing that cookie is instantly updated.

```jsx title='pages/index.js' lineNumbers
import { useCookie, useAction } from 'firebolt'

export default function Home() {
  const [theme, setLocalTheme] = useCookie('theme')
  const setServerTheme = useAction(updateTheme)
  return (
    <div>
      <p>Theme: {theme || 'system'}</p>
      <div onClick={() => setLocalTheme('light')}>Light</div>
      <div onClick={() => setLocalTheme('dark')}>Dark</div>
      <div onClick={() => setLocalTheme(null)}>System</div>
      <div onClick={() => setServerTheme('light')}>Light (Server)</div>
      <div onClick={() => setServerTheme('dark')}>Dark (Server)</div>
      <div onClick={() => setServerTheme(null)}>System (Server)</div>
    </div>
  )
}

export async function updateTheme(req, theme) {
  req.cookies.set('theme', theme)
}
```

The example above demonstrates how it doesn't matter where you modify a cookie, the UI will always reflect the truth.

To learn more about accessing cookies, check out the [useCookie](/docs/ref/useCookie) and [Request](/docs/ref/request) docs.
