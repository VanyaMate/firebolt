# firebolt.config.js

The Firebolt config file allows you to change the way your app is configured.

```jsx
export default function config() {
  return {
    // your config here
  }
}
```

## Port

By default your app will attempt to use the `PORT` environment variable, otherwise it will use the port in your config file which defaults to `3000`.

```jsx
export default function config() {
  return {
    port: 4000,
  }
}
```

## Source Maps

Production builds don't include source maps in client bundles by default, but you can enable them with this setting if needed:

```jsx
export default function config() {
  return {
    productionBrowserSourceMaps: true,
  }
}
```

## Middleware

Use middleware to decorate the [Request](/docs/ref/request) object that is passed to all [Loaders](/docs/loaders) and [Actions](/docs/actions):

```jsx
import db from './db'

export default function config() {
  return {
    middleware(req) {
      req.db = db
    },
  }
}
```

## MDX Plugins

By default MDX is not configured to use any plugins but you can configure them here to add additional behavior:

```jsx
import remarkGFM from 'remark-gfm'
import rehypeShiki from '@shikijs/rehype'

export default function config() {
  return {
    mdx: {
      remarkPlugins: [remarkGFM],
      rehypePlugins: [rehypeShiki],
    },
  }
}
```

## Cookie Defaults

When updating cookies in your app, if you don't provide any options they will expire after the current session by default.

To change this you can configure your default cookie options here instead of having to specify options everywhere else in your app.

```jsx
export default function config() {
  return {
    cookie: {
      expires: 30, // expire in 30 days
    },
  }
}
```

See [useCookie](/docs/ref/useCookie) for all available options.

## Public Env Prefix

By default any environment variables with the `PUBLIC_` prefix will be available on the client, but you can use a different prefix if needed:

```jsx
export default function config() {
  return {
    publicEnvPrefix: 'ANOTHER_PREFIX_',
  }
}
```

Learn more about environment variables on the [.env](/docs/ref/env) page.
